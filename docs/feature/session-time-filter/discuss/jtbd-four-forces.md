# Four Forces Analysis: Session Time Filter

## Job 1 — Focus on what's happening now

| Force | Description |
|-------|-------------|
| **Push** (frustration) | The sessions list shows every session ever recorded. As sessions accumulate, finding active ones requires visual scanning for the green dot among many rows. |
| **Pull** (desired future) | One click to see only active sessions. The list is short, relevant, and immediately actionable. |
| **Anxiety** (adoption concern) | Minimal. Filter is additive UI — doesn't remove existing functionality. Minor concern: "will I lose context about past sessions?" |
| **Habit** (current behavior) | Users visually scan for pulsing green dots. Some may scroll past completed sessions without reading them. |

## Job 2 — Review recent activity

| Force | Description |
|-------|-------------|
| **Push** (frustration) | After returning from a break, there's no way to scope the list to "what happened while I was gone." The full chronological list gives no temporal context beyond the timestamp on each row. |
| **Pull** (desired future) | Select "Last hour" and see exactly the sessions relevant to the current work period. Quick mental model of recent activity. |
| **Anxiety** (adoption concern) | "What if the time window cuts off a session I care about?" — sessions that started before the window but were active within it should still appear. |
| **Habit** (current behavior) | Users mentally note timestamps on session rows and estimate recency. Some may not bother reviewing older sessions at all. |

## Job 3 — Understand session volume over time

| Force | Description |
|-------|-------------|
| **Push** (frustration) | The "N total" count in the header reflects all-time sessions, not recent activity. No quick way to gauge "how busy was I today?" |
| **Pull** (desired future) | The count in the header reflects the selected time window, giving instant density feedback. |
| **Anxiety** (adoption concern) | None significant. This is passive information display. |
| **Habit** (current behavior) | Users may glance at the total count but it doesn't convey useful recency information. |
